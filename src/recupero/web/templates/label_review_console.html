🏷️ Label-candidate Review

Review the auto-ingested label-candidate queue. Paste your operator key to load; data is fetched directly from the admin-gated API (the key stays in a request header, never the URL).
This console is read-only — it displays the candidate queue. Promote / reject are mutating actions and are done via the API with the candidate-row confirm hash: POST /v1/labels/candidates/<id>/promote (with X-Recupero-Promote-Confirm) or POST /v1/labels/candidates/<id>/reject.
